# 13개 품질 지표 점수 보정 (Score Calibration)

## 배경

기존 13개 품질 지표 점수를 698건 데이터에 대해 검증한 결과, 세 가지 구조적 문제를 확인함.

### 문제 1: 짧은 상담 저평가

대화 턴 수가 적으면 키워드 절대량이 낮아 자동으로 저점이 산출됨.

**실제 사례 — 정수기 자동이체 카드 변경 (8턴)**
- 상담사가 6단계를 주도적으로 진행 (본인확인 → 주소확인 → 상품확인 → 계약확인 → 설치일 확인 → 카드 변경)
- 기존 주도성 점수: **7.5 / 100** — 명백한 저평가
- 기존 해결의지 점수: **23.5 / 100**
- 기존 다음단계명확성: **18.2 / 100**

| 지표 | 턴 8 (이 케이스) | 턴 20 (평균) | 원인 |
|---|---|---|---|
| 주도성 | 7.5 | 35.3 | 키워드 10개 / 턴 8 → 밀도 높지만 절대량이 적어 저점 |

### 문제 2: 분포 압축 (변별력 부재)

일부 지표의 점수가 좁은 범위에 몰려 좋고 나쁨을 구분하지 못함.

| 지표 | 평균 | 표준편차 | 실질 분포 | 문제 |
|---|---|---|---|---|
| 설명명확성 | 77.0 | **7.2** | 70~85에 몰림 | NPS 1~4(75.3)와 NPS 10(76.5) 차이 1.2점 |
| 협조도 | 80.1 | **13.1** | 70~95에 몰림 | CSAT 1~3(76.2)와 CSAT 5(80.2) 차이 4.0점 |

### 문제 3: NPS/CSAT 예측력 부재

13개 지표 종합점수와 외부 지표 간 상관이 거의 0.

| 종합점수 상관 | 보정 전 |
|---|---|
| → NPS | r = 0.007 |
| → CSAT | r = 0.041 |

## 보정 방법

### 모듈 위치

`analysis/score_calibrator.py` → `calibrate_scores(scores, stt)`

### 보정 1: 분포 재스케일링

압축된 점수 분포를 0~100 전체 범위로 펼침.

```
원래:  |-------[====]-------|  (70~85에 몰림)
보정:  |[================]--|  (10~95로 펼침)
```

| 대상 지표 | 원래 범위 | 재스케일 범위 | 결과 std |
|---|---|---|---|
| 설명명확성 | 60~90 → 10~95 | std 7.2 → **19.8** | 변별력 2.7배 |
| 협조도 | 40~100 → 10~95 | std 13.1 → **16.3** | 변별력 1.2배 |

### 보정 2: 길이 편향 보정

짧은 상담에서 키워드 **밀도(턴당 출현율)**가 높으면 점수를 부스트.

**적용 조건:**
- 상담사 턴 수 < 15 (짧은 상담)
- 해당 지표의 키워드 밀도 > threshold

**부스트 공식:**
```
turn_factor = 1 - (턴 수 / 15)          # 짧을수록 높음 (0~1)
density_factor = min(밀도 / threshold, 3) / 3  # 밀도 높을수록 높음 (0~1)
boost = max_boost × turn_factor × density_factor
```

**최소 하한선:**
- 키워드 밀도가 0.8 이상이면 최소 40점 보장
- 짧지만 밀도가 높은 상담이 10점 이하로 떨어지는 것을 방지

| 대상 지표 | 밀도 threshold | max_boost | 측정 키워드 예시 |
|---|---|---|---|
| 주도성 | 0.4 | 35 | 제가, 먼저, 확인해, 도와드리, 겠습니까 |
| 해결의지 | 0.5 | 30 | 확인, 조회, 처리, 진행, 접수, 변경 |
| 다음단계명확성 | 0.3 | 25 | 방문, 예약, 연락, 배송, 설치, 시간, 일 |
| 해결진척도 | (단계 기반) | 15/단계 | 확인→안내→처리→이행 4단계 도달 여부 |

### 보정 3: 보조 피처 블렌딩

기존 점수에 키워드 기반 보조 신호를 20% 가중 혼합.

| 대상 지표 | 보조 신호 | 블렌딩 비율 | 근거 |
|---|---|---|---|
| 협조도 | "네/예/알겠/감사" 턴당 밀도 | 기존 80% + 보조 20% | CSAT 상관이 0.069→0.092로 개선 |

## 파이프라인 적용

`analysis/batch_runner.py`의 `_serialize_result()`에서 Excel 점수를 읽은 직후 `calibrate_scores()`를 호출하여 보정된 점수를 JSON에 저장.

```
Excel 점수 읽기 → calibrate_scores(scores, stt) → JSON 저장 → 리포트 생성
```

## 한계 및 향후 과제

### 보정하지 않은 문제

| 문제 | 해당 지표 | 이유 |
|---|---|---|
| 키워드로 측정 불가능한 의미 | 문제구체성, 문제객관성 | BERT 수준의 문맥 이해 필요 |
| NPS 방향 역전 | 해결의지, 주도성, 다음단계 | NPS 1~4가 여전히 높게 나옴. 불만 고객일수록 상담이 길어져 키워드가 많아지는 구조적 원인 |
| NPS 자체의 한계 | 전체 | NPS는 상담 품질뿐 아니라 제품 문제 심각도에 크게 좌우됨 |

### 개선 방향

1. **BERT 기반 의미 채점** — 키워드 카운팅 대신 문맥 임베딩으로 주도성/구체성 판별
2. **상담 유형별 정규화** — 단순 문의(카드 변경)와 복합 문의(제품 고장)를 같은 기준으로 채점하지 않기
3. **NPS 예측 모델** — 13개 지표 + 오디오 피처를 결합한 회귀 모델로 가중치 최적화
