# sentiment-analysis

## 1. 현재 파이프라인 구조

```
WAV (GSM 8kHz mono)
  │
  ├─ Forced Alignment (stable-ts base) ──→ 단어별 실제 시각 (±0.1초)
  │     기존 STT 텍스트를 오디오에 강제 정렬
  │
  ├─ 음성 특징 추출 (librosa) ──→ 구간별 F0, Energy, ZCR, MFCC, Voiced Ratio
  │     baseline 보정: 해당 고객의 통화 내 평균 대비 상대적 변화로 Valence 산출
  │
  └─ STT 텍스트 ──→ BERT 60-class 감정 분류 ──→ 5개 상담 그룹으로 합산
        hun3359/klue-bert-base-sentiment       (라벨 개수 편향 보정 포함)
                    │                                    │
                    └───────── LLM 심판관 (Claude) ───────┘
                                    │
                    BERT 결과 vs 음성 결과 비교·검증
                    충돌 시 어느 쪽을 신뢰할지 근거 포함 판정
                                    │
                              최종 감정 그룹 + Valence + 근거
```

## 2. 각 모듈 상세

### 2.1 타임스탬프: Forced Alignment (stable-ts)

| 항목 | 내용 |
|------|------|
| 모델 | stable-ts (Whisper base 기반) |
| 방식 | 기존 STT 텍스트를 오디오에 단어 단위 강제 정렬 |
| 정확도 | 단어 단위 ±0.1초 |
| 이전 방식 | 문자 수 비례 배분 → 실제와 최대 수십 초 오차 |
| 처리 시간 | 콜당 ~5초 |

### 2.2 텍스트 감정: BERT 60-class → 5그룹

| 항목 | 내용 |
|------|------|
| 모델 | hun3359/klue-bert-base-sentiment (KLUE-BERT) |
| 학습 데이터 | GoEmotions 한국어 번역본 (Reddit 댓글 58K건) |
| 출력 | 60개 세분화 감정 라벨 확률 분포 |
| 재매핑 | 5개 상담 감정 그룹으로 합산 (라벨 개수 편향 보정 포함) |
| 대상 | 고객 발화만 (상담사 제외) |

**5개 상담 감정 그룹:**
- 감사/만족: 기쁨, 감사하는, 만족스러운, 신이 난, 안도, 자신하는 (6개 라벨)
- 안정/중립: 느긋, 편안한, 신뢰하는, 조심스러운, 남의 시선을 의식하는 (5개 라벨)
- 불안/걱정: 초조한, 걱정스러운, 불안, 안달하는 등 (10개 라벨)
- 불만/짜증: 짜증내는, 분노, 실망한, 슬픔, 비통한 등 (32개 라벨)
- 혼란/당황: 당황, 당혹스러운, 혼란스러운, 충격 받은, 흥분 (7개 라벨)

**라벨 편향 보정:**
부정 라벨이 48/60개로 과다 → 그룹별 라벨 수로 나누어 정규화 (라벨당 평균 확률)
보정 전: "네 이한성입니다" → 불만/짜증 37% (오판)
보정 후: "네 이한성입니다" → 감사/만족 34% (합리적)

### 2.3 음성 감정: baseline 보정 프로소디

| 항목 | 내용 |
|------|------|
| 추출 특징 | F0(피치), Energy(에너지), ZCR(제로교차율), MFCC(13차), Voiced Ratio |
| baseline | 해당 고객의 통화 내 전체 발화 평균을 기준점(0)으로 설정 |
| Valence | 평균 대비 z-score 기반 상대적 변화량 |

**baseline 보정이 필요한 이유:**
- 절대값 기준: "Energy 0.01 = 부정" → 원래 조용히 말하는 사람을 오판
- baseline 기준: "이 사람의 평균 Energy 0.025인데 이번 발화는 0.01 → 평소보다 위축" → 상대적 판단

**보정 전후 비교 예시:**

| 발화 | 보정 전 | 보정 후 | 해석 |
|------|--------|--------|------|
| "네 이한성입니다" | -0.134 (부정) | -0.037 (중립) | 평소 톤과 유사 |
| "네 네" (대기 후) | +0.180 (긍정) | -0.001 (중립) | 평소와 동일 |
| "네 수고하세요" | -0.241 (부정) | -0.425 (부정) | 평소보다 확실히 위축 |

### 2.4 LLM 심판관 (Claude API)

| 항목 | 내용 |
|------|------|
| 역할 | BERT 결과와 음성 결과를 비교·검증하여 최종 판정 |
| 입력 | BERT 그룹 확률 + 음성 Valence + 대화 맥락 |
| 출력 | 최종 감정 그룹 + Valence + 신뢰도 + 판정 근거 |
| 모델 | Claude Sonnet |

**LLM이 하지 않는 것:**
- STT 원문을 직접 읽고 감정 분석 (텍스트 편향 방지)
- 음성을 직접 듣고 판단 (불가능)

**LLM이 하는 것:**
- BERT가 "불안/걱정 60%"인데 음성이 "평소 톤(0.01)" → "텍스트는 걱정이지만 음성은 차분. 정보 확인 목적의 질문으로 판단" → 안정/중립
- BERT가 "안정/중립"인데 음성이 "평소보다 강한 부정(-0.44)" → "텍스트는 중립이지만 음성에서 위축 감지" → 불안/걱정

### 2.5 짧은 발화 처리

| 발화 유형 | 처리 방식 |
|----------|----------|
| 의미 있는 문장 (7자+) | BERT 텍스트 + 음성 특징 → LLM 심판 |
| 짧은 응답 ("네", "예 예") | 텍스트 분석 스킵, 음성 특징만으로 판정 → LLM 심판 |

## 3. 현재 구조의 강점

1. **멀티모달 융합**: 텍스트와 음성을 독립 분석 후 LLM이 교차 검증
2. **baseline 보정**: "원래 조용한 사람" vs "평소보다 위축된 사람" 구분
3. **실제 타임스탬프**: Forced Alignment로 발화 시각 ±0.1초 정확도
4. **라벨 편향 보정**: BERT의 60개 라벨 불균형(부정 48개) 구조적 보정
5. **LLM 심판관**: BERT와 음성이 충돌할 때 맥락 기반 판정 + 자연어 근거
6. **투명한 근거**: 모든 판정에 "왜 이 결과인지" 설명 포함

## 4. 현재 구조의 한계

### 4.1 음성 분석 정확도

- **현재**: 룰 기반 휴리스틱 (F0, Energy, ZCR → 가중합)
- **한계**: 실제 감정과 음향 특징의 관계가 단순 선형이 아님
  - 분노와 기쁨이 비슷한 음향 패턴 (높은 에너지, 높은 F0)
  - 개인 차이 (baseline 보정으로 일부 완화되나 불완전)
- **개선 방향**: wav2vec2/HuBERT 기반 pretrained SER(Speech Emotion Recognition) 모델 도입
  - 레이블 데이터 없이 사용 가능 (이미 학습된 모델)
  - 단, 한국어 pretrained SER 모델 품질이 일정하지 않음
  - 콜센터 도메인과 학습 데이터(드라마/일상 대화) 도메인 불일치 가능

### 4.2 BERT 모델 도메인 불일치

- **현재**: GoEmotions (Reddit 댓글) 기반 학습
- **한계**: SNS 감정 표현 ≠ 콜센터 상담 감정 표현
  - "비통한", "질투하는" 등 상담과 무관한 라벨이 확률에 혼입
  - 그룹 합산 + 편향 보정으로 완화하나 근본적 한계
- **개선 방향**: 콜센터 상담 데이터로 fine-tuning (레이블 데이터 필요)

### 4.3 mono 오디오 화자 분리 불가

- **현재**: STT의 [상담사]/[고객] 태그에 의존
- **한계**: WAV 파일이 mono → 음성 특징 추출 시 상담사/고객 음성이 섞일 수 있음
  - Forced Alignment이 발화 구간을 잡아주지만, 겹치는 발화(동시 발화)는 처리 불가
- **개선 방향**: Speaker Diarization (pyannote.audio) 또는 stereo 녹음 필요

### 4.4 LLM 비용 및 일관성

- **비용**: Claude API 호출당 약 $0.01~0.03 (콜당)
- **일관성**: 동일 입력에 대해 매번 약간 다른 판정 가능 (temperature 영향)
- **속도**: 콜당 ~3초 추가 지연
- **개선 방향**: temperature=0 설정, 또는 LLM 판정 결과를 캐싱

### 4.5 타임스탬프 한계

- **현재**: Forced Alignment (stable-ts base 모델)
- **한계**: mono 오디오에서 화자 경계 ±1~2초 오차 가능
  - 상담사 발화 끝과 고객 발화 시작이 겹치는 구간
  - 침묵 구간의 소속 (상담사 발화 끝? 고객 발화 시작?)
- **개선 방향**: Whisper large 모델 또는 stereo 녹음

## 5. 향후 개선 로드맵

| 우선순위 | 개선 항목 | 효과 | 필요 조건 |
|---------|----------|------|----------|
| 1 | 콜센터 도메인 BERT fine-tuning | 텍스트 분류 정확도 대폭 향상 | 레이블 데이터 500건+ |
| 2 | pretrained SER 모델 실험 | 음성 감정 정확도 향상 | 없음 (다운로드만) |
| 3 | Speaker Diarization | 화자별 음성 특징 분리 | HuggingFace 토큰 |
| 4 | Whisper large + Forced Alignment | 타임스탬프 정확도 향상 | GPU 권장 |
| 5 | LLM 판정 캐싱 | 비용 절감 + 일관성 확보 | 없음 |
